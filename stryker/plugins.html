    • Plugins

    Stryker supports several plugins, which are listed below. You can also search npm for the stryker-plugin tag.

    Don't worry about the reporters if you're just starting. The needed plugins will be installed for you if you're following out quickstart.

    Missing something? Feel free to create an issue or

    ðŸ”— Test runners

    The test runner plugin is able to hook into your test runner to improve performance. If your test runner isn't listed here you can probably still use the command test runner.

    • Jasmine (@stryker-mutator/jasmine-runner)
    • Jest (@stryker-mutator/jest-runner)
    • Karma (@stryker-mutator/karma-runner)
    • Mocha (@stryker-mutator/mocha-runner)

    Removed:

    • Web Component Tester (@stryker-mutator/wct-runner) (Removed in v4 of Stryker).

    ðŸ”— Reporters

    Stryker already has reporters build in:

    • progress: Report progress during mutation testing.
    • dots: Report progress as dots during mutation testing.
    • clear-text: A clear text report at the end of the mutation testing phase.
    • event-recorder: Write all events to disk. Useful for debug purposes (default location: 'reports/mutation/events')
    • html: An html report. See Stryker's own report for an example.
    • dashboard: Upload your mutation testing report to the Stryker dashboard. See the Stryker handbook for more info.
